# 📝 文章归档系统

这个文件夹存放与cc共同创作的文章。

## 归档规则

### 文件命名规范
```
YYYY-MM-DD_文章标题关键词.md
```

示例：
- `2026-01-28_MyCC-国产版clawdbot的接地气选择.md`
- `2026-02-15_技术趋势分析.md`

### 文章模板
每篇文章应包含：
1. **标题** + 副标题（可选）
2. **生成日期**和**作者**信息
3. **正文内容**
4. **文章信息**（字数、关键词、目标读者、发布平台）
5. **后记**（可选，记录创作过程）

### 元数据管理
在文章开头使用YAML frontmatter记录元数据：
```yaml
---
title: 文章标题
date: YYYY-MM-DD
author: 作者信息
tags: [标签1, 标签2]
category: 分类
word_count: 字数
target_audience: 目标读者
platforms: [发布平台1, 发布平台2]
---
```

## 分类系统

### 按主题分类
1. **技术推荐** - 工具、软件、开源项目推荐
2. **效率方法** - 工作流、时间管理、生产力技巧
3. **行业观察** - 技术趋势、市场分析、行业洞察
4. **个人成长** - 学习、职业发展、技能提升
5. **产品评测** - 实际使用体验和评测

### 按读者分类
- **技术人群** - 开发者、工程师、技术管理者
- **上班族** - 普通职员、管理者、自由职业者
- **学生群体** - 大学生、研究生、自学者
- **创业者** - 初创团队、独立开发者

## 创作流程

### 1. 选题讨论
- 确定文章主题和目标读者
- 讨论文章角度和核心观点
- 预估文章长度和结构

### 2. 大纲制定
- 创建详细文章大纲
- 确定各部分内容和比例
- 确认关键案例和数据点

### 3. 内容创作
- 基于大纲撰写正文
- 保持语言风格一致
- 避免AI味，保持自然

### 4. 修订优化
- 检查逻辑连贯性
- 优化语言表达
- 添加具体案例和数据
- 调整段落节奏

### 5. 归档发布
- 按规范命名文件
- 添加元数据信息
- 更新README文章列表
- 准备发布版本

## 质量标准

### 内容质量
- ✅ **观点明确** - 有清晰的核心观点
- ✅ **案例具体** - 使用真实或合理的案例
- ✅ **逻辑连贯** - 文章结构合理，逻辑清晰
- ✅ **语言自然** - 避免AI味，保持人类写作风格
- ✅ **价值突出** - 对读者有实际帮助

### 技术规范
- ✅ **格式规范** - 符合Markdown语法
- ✅ **命名规范** - 按日期+关键词命名
- ✅ **元数据完整** - 包含必要的元信息
- ✅ **版本管理** - 通过Git进行版本控制
